# ESP32 三顆共陰 RGB LED 接線指南

本文件適用於 ESP32 32E (WROOM) 與三顆 4 腳共陰 RGB LED。三顆 LED 分別代表前、中、後三個區域。

## 準備材料

- 1 個 ESP32 32E (WROOM)
- 3 顆 4 腳共陰 RGB LED
- 建議：9 顆 220-330 ohm 限流電阻，每個顏色通道各一顆
- 臨時接法：至少 3 顆 220-330 ohm 限流電阻，每顆 LED 的共陰腳各一顆
- 麵包板與跳線

## GPIO 對照表

| 區域 | 程式索引 |  紅色 R |  綠色 G |  藍色 B | 共陰腳       |
| ---- | -------: | ------: | ------: | ------: | ------------ |
| 前排 |        0 | GPIO 25 | GPIO 26 | GPIO 27 | 接至 GND     |
| 中排 |        1 | GPIO 32 | GPIO 33 | GPIO 14 | 接至 GND     |
| 後排 |        2 | GPIO 18 | GPIO 19 | GPIO 21 | 接至 GND     |

GPIO 配置定義於 `esp32/strip.py` 與 `esp32-build/strip.py` 的 `LED_PINS`。

## 建議接線方式（9 顆電阻）

每個 R、G、B 腳都必須獨立串接一顆限流電阻。三顆 LED 的共陰腳全部接到 ESP32 的 GND。

```text
ESP32 GPIO ── 220-330 ohm 電阻 ── LED 顏色陽極
ESP32 GND  ────────────────────── LED 共陰腳
```

### 前排 LED

```text
GPIO 25 ── 電阻 ── R
GPIO 26 ── 電阻 ── G
GPIO 27 ── 電阻 ── B
GND     ────────── 共陰腳
```

### 中排 LED

```text
GPIO 32 ── 電阻 ── R
GPIO 33 ── 電阻 ── G
GPIO 14 ── 電阻 ── B
GND     ────────── 共陰腳
```

### 後排 LED

```text
GPIO 18 ── 電阻 ── R
GPIO 19 ── 電阻 ── G
GPIO 21 ── 電阻 ── B
GND     ────────── 共陰腳
```

## 臨時接線方式（只有 3 顆電阻）

只有 3 顆 220-330 ohm 電阻時，可以讓每顆 LED 的共陰腳各串接一顆電阻後再接 GND。R、G、B 三個陽極仍須分別接到指定 GPIO，絕對不能互相短接。

```text
GPIO R ────────────────── R 陽極
GPIO G ────────────────── G 陽極
GPIO B ────────────────── B 陽極
LED 共陰腳 ── 電阻 ───── GND
```

三顆 LED 都使用相同方式：

```text
前排 LED 共陰腳 ── 電阻 ── GND
中排 LED 共陰腳 ── 電阻 ── GND
後排 LED 共陰腳 ── 電阻 ── GND
```

此接法只適合目前紅、綠、藍單色狀態。多個顏色通道同時亮起時會共用同一顆電阻，造成亮度與混色不準，因此不建議作為正式接法。

## 辨識 RGB LED 腳位

4 腳 RGB LED 的實際 R、G、B 排列會因廠牌而異，不要只依照網路圖片接線。

1. 優先查看 LED 賣場規格或資料表。
2. 最長腳通常是共用腳，但仍應以資料表或萬用電表確認。
3. 共陰 LED 的共用腳接 GND；其餘三腳是 R、G、B 的陽極。
4. 如果顏色與程式不一致，關閉電源後交換對應的顏色線，不要移動共陰腳。

## 重要注意事項

- 接線前先拔除 ESP32 的 USB 電源。
- 正式接法應讓每個顏色腳都有自己的限流電阻；只有 3 顆電阻時，才暫時改接在三顆 LED 各自的共陰腳。
- 不可在沒有任何限流電阻的情況下，將共陰腳直接接到 GND。
- 不要把 R、G、B 三個顏色腳接在一起，也不要接到 5V 或 3.3V。
- 共陰腳只能接 GND；如果買到共陽 RGB LED，此接法與目前程式不適用。
- 三顆 LED 可共用 ESP32 的 GND，但每個 R、G、B 通道必須使用各自指定的 GPIO。
- 3 顆電阻接法不適合同時驅動多個顏色通道，因此 fallback 已改為只使用藍色通道呼吸。

## 輕量測試

ESP32 與電腦連上相同網路後，在電腦執行：

```bash
uv run python scripts/check_led_strip_tcp.py
```

程式會監聽 TCP port `8765`，每 5 秒切換一次顏色：

1. 前排紅、中排綠、後排藍
2. 前排綠、中排藍、後排紅
3. 前排藍、中排紅、後排綠
4. 三顆全部熄滅

終端顯示 `connected: True` 代表 ESP32 已連上測試伺服器。按 `Ctrl-C` 停止測試。

## 故障排查

| 現象               | 檢查項目                                                          |
| ------------------ | ----------------------------------------------------------------- |
| LED 完全不亮       | 共陰腳是否接 GND、ESP32 是否收到 TCP 狀態、限流電阻與跳線是否接通 |
| 顏色錯誤           | R、G、B 實體腳位可能與資料表或接線不同                            |
| 只有一種顏色不亮   | 該顏色的電阻、跳線、LED 腳位或 GPIO 接錯                          |
| 三顆同步藍色呼吸   | ESP32 已進入 fallback，表示 10 秒內未收到電腦資料                 |
| `connected: False` | 確認電腦 IP、port 8765、防火牆，以及 ESP32 與電腦是否在相同網路   |
